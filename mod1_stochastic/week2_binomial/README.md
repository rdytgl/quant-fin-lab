# Weekly Summary
## 1. What did I model?
A binomial tree model is a simple model that maps out the two possible paths a stock price could go, either up or down by a certain rate of u and d. I created my first simulation for the one-step binomial tree model. Building from that code, I was able to build a two-step binomial tree model. Finally, with a deeper understanding on how the model works, I am able to create an n-step binomial tree pricer that allows me to input a given n step binomial tree given that all parameters (u, d, r, dt) are consistent for all steps.

## 2. What is the key insight?
One key insight is that a risk-less portfolio must earn exactly the risk-free rate — no more, no less. If an option is mispriced, free money exists and arbitrageurs correct it immediately. This pins the option to exactly one fair price. Risk-neutral valuation then simplifies pricing further by replacing unknown real-world probabilities with risk-neutral probabilities, where everything earns the risk-free rate — giving us a known discount rate always. This is also why drift (mu) disappears in options pricing. In the risk-neutral world, every asset grows at the risk-free rate regardless of its real-world expected return.

## 3. What do/es the code/s do?
The [first code](one_step_bt_pricer.py) is the base code for the onestep binomial tree pricer. The [second code](two_step_bt_pricer.py) extends the one-step model to two steps, folding back (or discounting back) from expiry to today twice using the same risk-neutral formula at each node. Lastly, the [third code](n_step_bt_pricer.py) generalises the fold-back loop for however many n-times is needed in the binomial tree model.

## 4. What surprised me?
I finally understood what building a risk-less portfolio means in practice. By going long 0.25 shares and short one call option, the gains and losses offset each other perfectly regardless of where the stock goes. Any mispricing of the option breaks this balance and creates arbitrage — which is exactly why options have one specific fair price.

Connecting from [week 1's reflections](../week1_gbm/WK1_README.md), I now also understand why drift doesn't matter as much in options trading, as the final price (and probabilities surrounding it) is not as important as ensuring that the option price is correctly figured out. This helps us achieve a risk-less portfolio that prevents arbitrage. Even if in the real world, investors are risk averse, the risk-neutral model works out still because we know that the payoff should be at the risk-free rate. What surprised me most was that real-world probabilities are completely irrelevant to pricing. You don't need to know the actual probability of the stock going up — the risk-neutral probability does all the work.
